本番環境のセキュリティ対策

公開後は攻撃対象になる可能性があります。最低限の対策を確認しましょう。

💉
SQLインジェクション対策
❌ db.prepare(`SELECT * FROM users WHERE email='${email}'`)
✅ db.prepare('SELECT * FROM users WHERE email=?').get(email)
🔑
パスワードハッシュ化
❌ db.run('INSERT INTO users VALUES (?)', password)
✅ const hash = bcrypt.hashSync(password, 10)
db.run('INSERT INTO users VALUES (?)', hash)
🌐
CORS設定（本番用）
❌ app.use(cors()) // 全てのオリジンを許可
✅ app.use(cors({ origin: process.env.CORS_ORIGIN }))
// 本番フロントのURLだけ許可
🛡️
Helmetでヘッダー保護
✅ npm install helmet
app.use(helmet())
// XSS・クリックジャッキング等を防ぐ
⚡
レートリミット（ブルートフォース対策）
❌ ログインAPIに制限なし（何度でも試せる）
✅ npm install express-rate-limit
app.use('/api/auth', rateLimit({ max: 5 }))
🔒
環境変数の管理
✅ JWT_SECRET は 32文字以上のランダム文字列
✅ .env は絶対にGitにpushしない
✅ デプロイ先のEnv設定を使う
セキュリティは「後回し」厳禁！特に SQL インジェクション対策は今すぐ確認しよう
