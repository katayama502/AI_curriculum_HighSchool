関数 — 処理を「部品」にする

関数の宣言と呼び出し

// 関数の定義
function greet(name) {
  return `こんにちは、${name}さん！`;
}

// 関数の呼び出し
const message = greet("颯斗");
console.log(message);
// → "こんにちは、颯斗さん！"
関数の解剖
function関数を定義するキーワード
greet関数の名前（動詞で命名するとわかりやすい）
name引数（関数に渡す値）。複数もOK
return関数の戻り値。これが呼び出し元に返る

アロー関数（現代的な書き方）

// 従来の書き方
function add(a, b) {
  return a + b;
}

// アロー関数（同じ意味）
const add = (a, b) => a + b;

// コールバック関数として使う
[1, 2, 3].map(x => x * 2);
// → [2, 4, 6]
なぜ関数を使うのか？
① 同じ処理を何度も書かなくてよい（DRY原則）
② 名前をつけることで「何をする処理か」が分かる
③ テスト・修正が1箇所でできる
