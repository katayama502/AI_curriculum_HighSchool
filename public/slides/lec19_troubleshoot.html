デプロイのトラブルシューティング

503
Application error / Service Unavailable
原因: start スクリプト未設定、PORT 環境変数なし
✅ package.json の "start" コマンドと PORT=process.env.PORT を確認
CORS
Access-Control-Allow-Origin エラー
原因: CORS_ORIGIN が localhost のまま
✅ Railway 環境変数 CORS_ORIGIN を Vercel URL に更新
ENV
API URL が undefined / リクエストが localhost に飛ぶ
原因: VITE_API_URL 環境変数が Vercel に設定されていない
✅ Vercel → Settings → Environment Variables で設定してRedeploy
BUILD
Build failed / npm install error
原因: package-lock.json が無い、node_modules がコミットされている
✅ .gitignore に node_modules を追加して再push
DB
SQLITE_CANTOPEN / データが消える
原因: サーバーレス環境はファイルシステムが永続しない
✅ 本番では PlanetScale / Supabase / Railway MySQL に移行
404
Page not found on refresh (SPA)
原因: React Router のルートを Vercel が認識しない
✅ vercel.json に rewrites 設定を追加
// vercel.json (React Router の 404 対策)
{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
エラーが出たら: ① ログを読む → ② エラーメッセージをコピー → ③ Claude に貼り付けて「何が原因か教えて」
