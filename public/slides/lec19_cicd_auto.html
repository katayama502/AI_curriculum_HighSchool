CI/CDパイプラインの自動化

自動デプロイの流れ

コードを修正
→
git push main
→
テスト自動実行
→
本番デプロイ
# .github/workflows/deploy.yml
name: CI/CD
on:
  push:
    branches: [main]

jobs:
  test-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npm test # テスト実行
      - # PASSならVercel/Railwayが自動デプロイ

自動化のメリット

✅
毎回手動でデプロイする手間が省ける
✅
テストが失敗したらデプロイが止まる（壊れた状態で公開されない）
✅
プルリクエストごとにプレビュー環境が作られる
✅
チーム開発でも同じ手順で安全にリリースできる
🚀
Vercel・Railwayは設定なしでGitHub連携自動デプロイが完成している
🚀
GitHub Actionsで更に細かいカスタマイズが可能
