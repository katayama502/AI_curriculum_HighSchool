APIテスト：Supertest で統合テスト

Express API を丸ごとテスト

// npm install --save-dev jest supertest
const request = require('supertest');
const app     = require('./app'); // Express アプリをexport

describe('GET /api/todos', () => {
  test('200を返してリストを取得できる', async () => {
    const res = await request(app).get('/api/todos');
    expect(res.status).toBe(200);
    expect(Array.isArray(res.body)).toBe(true);
  });
});

describe('POST /api/todos', () => {
  test('新しいToDoを作成して201を返す', async () => {
    const res = await request(app)
      .post('/api/todos')
      .send({ text: 'テスト用ToDo' });
    expect(res.status).toBe(201);
    expect(res.body).toHaveProperty('id');
  });

  test('textが空の場合400を返す', async () => {
    const res = await request(app)
      .post('/api/todos').send({ text: '' });
    expect(res.status).toBe(400);
  });
});

テストのコツ

テスト用DBを分ける
NODE_ENV=test 時はインメモリDBを使うと、本番データを汚さずテストできる。
beforeEach/afterEach
各テスト前後にDBリセットする処理を書くと、テスト間の干渉を防げる。
カバレッジ確認
jest --coverage でどのコードがテストされているか割合で確認できる。
