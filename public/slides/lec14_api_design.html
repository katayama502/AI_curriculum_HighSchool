REST API 設計：ToDoアプリのエンドポイント

/api/todos のCRUD設計

メソッド パス 説明 リクエスト例 レスポンス例
GET /api/todos 全件取得 — [{id:1, text:"学習", done:0}]
POST /api/todos 新規作成 {text: "Reactを学ぶ"} {id: 2, text:"Reactを学ぶ"}
PUT /api/todos/:id 完了切り替え {done: 1} {id:1, done:1}
DELETE /api/todos/:id 削除 — {message:"削除しました"}
URL設計のルール
名詞を使う（/todos ○ / /getTodos ×）。複数形にする。IDは /:id で指定。
テストツール
APIの動作確認には Bruno や Thunder Client（VSCode拡張）を使うと便利。
