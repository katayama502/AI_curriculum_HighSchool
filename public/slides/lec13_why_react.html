なぜReactを使うのか？

バニラJSとReactの比較

✗ バニラJS — 手動でDOMを操作
// ToDoを追加するたびに…
const li = document.createElement('li');
li.textContent = newTodo;
li.className = 'todo-item';
document.querySelector('#list').appendChild(li);
// 削除・更新も全部手動で書く
コードが長くなりバグが増える
データとUIの同期が大変
✓ React — データを変えるだけでUIが自動更新
// データを更新するだけ！
const [todos, setTodos] = useState([]);

function addTodo(text) {
setTodos([...todos, text]);
// UIは自動で更新される！
}
データとUIが自動で同期する
部品（コンポーネント）を再利用できる

Reactを学ぶ3つの理由

🏢
業界標準のライブラリ
Meta（Facebook）が開発。Instagram・Facebook・Airbnbなど世界中のサービスで使用されている
🧩
コンポーネント設計
UIをレゴブロックのように組み合わせる考え方。一度作ったパーツを何度でも再利用できる
🤖
AIとの相性が抜群
Claude Codeで「〇〇コンポーネントを作って」と指示するだけで高品質なコードが生成できる
