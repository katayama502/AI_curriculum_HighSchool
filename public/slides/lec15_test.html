テストの基礎知識

🧪 なぜテストが必要か？
🛡️
バグを早期に発見できる
本番環境に出る前に問題を検出
🔄
リファクタリングが安全になる
変更しても既存機能が壊れていないか確認
📚
コードの仕様書になる
テストを読めば何をするコードか分かる
🤝
チーム開発の品質を保つ
誰かの変更が他に影響しないか検証
💻 Jestでユニットテスト
// テスト対象の関数
function add(a, b) { return a + b; }

// テストコード
describe('add関数', () => {
  test('1 + 2 は 3 になる', () => {
    expect(add(1, 2)).toBe(3);
  });

  test('負の数も計算できる', () => {
    expect(add(-1, -2)).toBe(-3);
  });
});

// $ npx jest → ✓ 2 tests passed
✅ Claudeに「このコードのテストを書いて」と頼もう！
