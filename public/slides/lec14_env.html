環境変数：秘密情報を安全に管理

.env ファイルの使い方

# .env ファイル（GitHubに上げない！）
PORT=3000
DB_PATH=./data.db
API_KEY=sk-xxxxxxxxxxxx
JWT_SECRET=mysecretkey
// index.js で読み込む
require('dotenv').config();

// process.env で参照
const port = process.env.PORT || 3000;
const apiKey = process.env.API_KEY;
NG — コードにAPIキーを直書き
GitHubに上げた瞬間に世界中に漏れる！クレジットカード不正利用の被害事例も多数あり。絶対にやめよう。
OK — .env ファイルで管理
.gitignore に .env を追記すれば、ローカルにしか存在しないシークレット情報として安全に管理できる。
.env.example を用意しよう
キー名だけ書いた .env.example をGitに含めると、他の人がどんな変数が必要かわかる。
