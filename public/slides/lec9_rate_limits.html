レート制限とAPI利用の注意点

レート制限（Rate Limit）とは？

リクエスト数の上限

多くのAPIは「1分間に60回まで」などリクエスト数に上限があります。超えると429エラーが返ってきます。

対処法：少し待ってリトライ

429が返ってきたら一定時間待ってから再送する処理を実装します。

import time, requests for i in range(10): res = requests.get(url) if res.status_code == 429: print("制限に達した。1秒待ちます") time.sleep(1) continue # 正常処理 data = res.json() time.sleep(0.1) # 少し間隔を空ける

API利用のベストプラクティス

  • APIキーは環境変数に保存してコードに直書きしない

  • エラーハンドリングを必ず実装。ネットワーク障害や503に備える

  • 不要なリクエストを減らす。同じデータは一度だけ取得してキャッシュする

  • 利用規約を確認する。商用利用禁止・クレジット表記が必要なAPIもある

  • 無料枠の上限に注意。超えると課金されるAPIは使用量を監視する
