JSONを読む・使う

APIが返すデータの形式

// 天気APIのレスポンス例（JSON） { "city": "東京", "temperature": 22, "condition": "晴れ", "forecast": [ { "day": "月", "temp": 20 }, { "day": "火", "temp": 18 } ] } // JSでの使い方 const data = await res.json(); console.log(data.city); // "東京" console.log(data.forecast[0].day); // "月" # Pythonでの使い方 data = res.json() print(data["city"]) # 東京 print(data["forecast"][0]) # {'day':'月','temp':20}

JSONの書き方ルール

キーは必ず"ダブルクォート"で囲む
シングルクォートはNG
配列は [ ]、オブジェクトは { }
入れ子にもできる
末尾カンマはNG
最後の要素の後にカンマを付けない
使える値の型
文字列・数値・真偽値・null・配列・オブジェクト

💡 JSONはJavaScriptのオブジェクトに似ているが、別の形式。JSではJSON.parse()で変換する。Pythonでは辞書として扱える。
