今回の開発から学んだこと

計画が8割
最初にClaude Codeと設計を相談したことで、実装がスムーズに進んだ。「まず動くものを作る」という方針が大切。
小さいcommitが安全
機能ごとにcommitしていたので、複数都市機能で問題が起きた時もすぐ前の状態に戻せた。
バグは必ず起きる
福岡の座標が間違っていてデータが取れなかった。エラーをそのままAIに渡したら5秒で原因がわかった。
公開がモチベーションになる
GitHub PagesでURLが発行されて友達に送れた。「誰かに使ってもらえる」という体験がプログラミングの楽しさ。
