Claude Code × Git — 実践ワークフロー

AIと開発する時のGitフロー

1
機能ブランチを作る
git checkout -b feature/dark-mode
mainブランチを守るために、機能ごとにブランチを切る
2
Claude Codeで実装する
claude
AIと対話しながら機能を実装。コミットはAIに頼んでもOK
3
こまめにcommitする
git add . && git commit -m "feat: ダークモード追加"
小さな単位でcommit。「git add .」の前にgit statusで確認
4
mainにマージする
git checkout main && git merge feature/dark-mode
テストが通ったらmainにマージ。GitHubならPRを使う
良いcommitメッセージの書き方
✗ 悪い例
修正
いろいろ変えた
test
作業中
✓ 良い例（何をしたか一目瞭然）
feat: ダークモード切り替え機能を追加
fix: 大阪の緯度経度が間違っていたのを修正
style: ヘッダーのレスポンシブ対応
docs: README.mdにインストール手順を追記
