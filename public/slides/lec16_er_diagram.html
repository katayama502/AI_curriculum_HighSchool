1人のユーザーが多くの本を持つ = 1対多（1:N）の関係。FK（外部キー）でテーブルを繋ぐ。

設計のポイント

PRIMARY KEY (PK)
各テーブルの「ユニーク識別子」。id INTEGER PRIMARY KEY AUTOINCREMENT で自動付番。
FOREIGN KEY (FK)
他テーブルのPKを参照する列。user_id が users テーブルの id を参照する。
関係の種類
1対1（ユーザー→プロフィール）、1対多（ユーザー→投稿）、多対多（タグ↔記事）がある。
