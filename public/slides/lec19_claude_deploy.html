Claude Code でデプロイを加速

デプロイで詰まったとき・設定ファイルを作るとき、Claude Code に直接聞こう

PATTERN 1 — デプロイ前チェック
「本番デプロイ前に確認すべき
チェックリストを作って。
フロント: React + Vite
バックエンド: Express + SQLite」
PATTERN 2 — エラー解析
「Vercelのデプロイログにこのエラーが
出ています。原因と修正方法を教えて:
[ここにエラーメッセージを貼り付け]
PATTERN 3 — 設定ファイル生成
「GitHub Actions で push のたびに
Vercel に自動デプロイする
workflow ファイルを作って」
PATTERN 4 — セキュリティ診断
「このExpressのコードに
セキュリティ上の問題はある?
[server.js の内容を貼り付け]
本番公開前に修正したい」
エラーログは必ず全文コピーして貼る
設定ファイルは生成後に内容を理解する
修正後は必ず動作確認してからpush
