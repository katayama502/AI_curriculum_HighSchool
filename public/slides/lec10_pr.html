プルリクエスト（PR）とGitHub Flow

プルリクエストの流れ

1

ブランチを作成してコードを書く

feature/login などの名前でブランチを切る

2

GitHubにpushする

git push origin feature/login でアップロード

3

Pull Requestを作成

GitHub上でマージしてほしいことを伝える。変更内容の説明も書く。

4

コードレビュー

チームメンバーがコードを確認し、コメントや修正依頼を出す

5

承認されたらマージ

mainブランチに変更が取り込まれる

GitHub Flowまとめ

GitHub Flow（シンプルな開発手順）

mainブランチは常にデプロイできる状態を保つ

新機能・修正はfeatureブランチで作業する

定期的にpushしてチームと共有する

Pull Requestでレビューを依頼する

承認されたらmain にマージしてデプロイ

💡 1人でも GitHub Flow を使おう！「自分PRを自分でレビュー」でもコードの質が上がります。
