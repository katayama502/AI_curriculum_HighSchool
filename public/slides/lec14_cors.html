CORS：ポートが違うと弾かれる！

なぜ CORS エラーが起きるか

React と Express は別ポートで動いている
React
:5173
Express
:3000
Access to fetch at 'http://localhost:3000'
from origin 'http://localhost:5173'
has been blocked by CORS policy. 
const cors = require('cors');

// 開発中：Reactのポートを許可
app.use(cors({
  origin: 'http://localhost:5173'
}));

CORSとは？

Cross-Origin Resource Sharing
ブラウザには「異なるオリジン（ドメイン・ポート）へのリクエストは危険かも」という安全機能があります。

サーバー側で「このオリジンは許可する」と明示することで、React ↔ Express間の通信が可能になります。
本番環境では？
フロントとバックが同一オリジンになるか、本番URLを指定する。
origin: process.env.FRONTEND_URL
開発の便利設定： Vite の vite.config.js でプロキシを設定する方法もある。
