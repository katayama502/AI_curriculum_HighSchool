コンフリクト（競合）の解消

コンフリクトとは？

同じファイルの同じ行を2人が別々に変更すると、Gitがどちらを採用すべきかわからなくなった状態。

# コンフリクトが起きたファイルの中身 <h1>ようこそ</h1> <<<<<<< HEAD <p>最新のAI技術を学ぼう</p> ======= <p>プログラミングを一緒に学ぼう</p> >>>>>>> feature/top-copy # 解消後：手動でどちらか（または両方）を採用 <p>AIとプログラミングを一緒に学ぼう</p> # 解消後のコマンド $ git add . $ git commit -m "コンフリクトを解消"

コンフリクト解消の手順

1

コンフリクトしたファイルを開く

git statusで確認。`<<<<<<<`のマーカーを探す

2

どちらを採用するか決める

自分の変更・相手の変更・両方を組み合わせるの3択

3

マーカー行を全て削除する

<<<, ===, >>>の行を消して最終的な内容だけ残す

4

git add → git commit

解消したファイルをステージしてコミットする

💡 コンフリクトが怖いときは、Claudeにコンフリクトしたコードを貼って「どちらを採用すべきか」聞くと解決策を提案してもらえます！
