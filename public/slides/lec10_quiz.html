確認クイズ — Git・GitHub

Q1
変更したファイルをコミット前にステージに追加するコマンドは？
A. git commit
B. git add . ✓
C. git push
D. git stage
git addでファイルをステージへ、git commitで記録する2ステップが基本。
Q2
新しいブランチを作りながら移動するコマンドは？
A. git branch new-feature
B. git checkout -b new-feature ✓
C. git create new-feature
D. git switch --new new-feature
-bオプションで「作成＋移動」が一度にできる。git switch -cも同じ動作。
Q3
ローカルの変更をGitHubに送るコマンドは？
A. git send origin main
B. git push origin main ✓
C. git upload origin main
D. git commit origin main
pushでローカル→リモート（GitHub）へ送信。pullはリモート→ローカルへの取得。
Q4
.gitignoreファイルに記載すべきでないものは？
A. APIキーが入った.envファイル
B. node_modules/フォルダ
C. index.htmlやmain.pyなどのソースコード ✓
D. .DS_Store（macOSのシステムファイル）
ソースコード（書いたコード）こそGitで管理すべきもの。環境・依存・機密情報は除外する。
