手動テストと統合確認

REST Clientでエンドポイントをテスト

VS Code拡張「REST Client」を使うと、APIをコードから直接テストできる。curlより書きやすく、チームで共有できる。

### test.http ファイルに書く ###

### 1. 登録
POST http://localhost:3000/api/auth/register
Content-Type: application/json

{ "email": "test@example.com",
"password": "password123" }

### 2. ログイン → レスポンスのtokenをコピー
POST http://localhost:3000/api/auth/login
Content-Type: application/json

{ "email": "test@example.com",
"password": "password123" }

### 3. 本を追加（Bearerトークンを貼る）
POST http://localhost:3000/api/books
Authorization: Bearer {{token}}
Content-Type: application/json

{ "title": "React入門" }

統合テストチェックリスト

✅
ログイン→トークン取得→保護されたAPIが動く
✅
他人のデータにアクセスできないことを確認
✅
空のタイトルで400エラーが返る
✅
トークンなしで401エラーが返る
✅
フロントからCRUD全操作が動く
✅
ページリロードしてもログイン状態が維持される
✅
Networkタブでリクエスト/レスポンスが正常
