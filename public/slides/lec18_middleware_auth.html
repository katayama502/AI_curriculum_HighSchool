認証ミドルウェアの仕組み

リクエスト処理の流れ

クライアント
→
authMiddleware
→
ルートハンドラ
→
レスポンス
// middleware/auth.js
const jwt = require('jsonwebtoken');

module.exports = (req, res, next) => {
const header = req.headers.authorization;
if (!header) {
return res.status(401)
.json({ error: 'トークンが必要です' });
}
const token = header.split(' ')[1];
try {
const decoded = jwt.verify(token,
process.env.JWT_SECRET);
req.user = decoded; // req.user.userId で使える
next(); // 次の処理へ
} catch {
res.status(401).json({ error: '無効なトークン' });
}
};

ミドルウェアの使い方

個別ルートに適用： router.get('/', authMiddleware, handler) — そのルートだけ保護
ルーター全体に適用： router.use(authMiddleware) — 以降のルート全てに適用
req.user に情報が入る： req.user.userId でどのユーザーのリクエストか判定できる
注意： ログイン・登録エンドポイントには authMiddleware を付けない！
next() を呼ぶ＝「次のミドルウェアへ進む」。呼ばないとリクエストがストップする
