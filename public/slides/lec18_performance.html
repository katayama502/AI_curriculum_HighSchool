パフォーマンス改善の基本

「動く」が完成したら次は「速い」を目指す。基本のチューニングを覚えよう。

🔍
DBにインデックスを付ける
よく検索するカラムにINDEXを作ると検索が劇的に速くなる
CREATE INDEX idx_user_id ON books(user_id);
📦
React.memoで再描画を防ぐ
propsが変わっていないコンポーネントの再描画をスキップ
export default React.memo(BookCard);
🖼️
画像を最適化する
WebP形式を使い、サイズを圧縮。loading="lazy"で遅延読み込み
<img loading="lazy">
⚡
APIレスポンスをキャッシュ
変わらないデータは再取得しない。useMemoやReact Queryが有効
const data = useMemo(()=>..., []);
🗜️
バンドルサイズを小さくする
Viteのビルドで自動的に最小化される。不要なライブラリを削除
npm run build
📊
Lighthouseで計測する
Chrome DevTools内蔵。パフォーマンス・SEO・アクセシビリティを計測
DevTools → Lighthouse → 分析
「このAPIが遅い。どうすれば速くなりますか？」とClaude Codeに相談するのが最短ルート！
