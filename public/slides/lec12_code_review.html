AIによるコードレビュー

AIがよく指摘する問題点

セキュリティ問題
APIキーがハードコードされている
→ .envファイルに移動してprocess.env.API_KEYで参照
エラーハンドリング不足
APIが失敗したときの処理がない
→ try/catchで例外処理を追加、ユーザーにエラーを伝える
コードの重複（DRY違反）
同じ処理が3箇所に書かれている
→ 共通関数にまとめてコードを短くする
パフォーマンス問題
毎回APIを呼んでいて遅い
→ キャッシュを使って不要なリクエストを減らす

レビュー依頼プロンプト

// 包括的なレビュー依頼
script.jsを以下の観点でレビューしてください：

1. セキュリティ（APIキーの扱い、XSSなど）
2. エラーハンドリング（例外処理の漏れ）
3. コード品質（重複・読みやすさ）
4. パフォーマンス（無駄な処理がないか）

問題があれば修正コードも一緒に示してください。
レビュー前のセルフチェック
APIキーを直接書いていないか
全てのAPIコールにtry/catchがあるか
変数名・関数名はわかりやすいか
コピペコードがないか（DRY原則）
空のcatchブロックがないか
commitする前にブラウザで動作確認したか
