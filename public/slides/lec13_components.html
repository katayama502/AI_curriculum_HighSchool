コンポーネント設計

🧩 コンポーネントの分割方法
// 親コンポーネント
function App() {
  return (
    <div>
      <Header title="マイアプリ" />
      <MainContent />
      <Footer />
    </div>
  );
}

// 子コンポーネント（propsを受け取る）
function Header({ title }) {
  return <h1>{title}</h1>;
}
📋 コンポーネント設計の原則
🎯
1コンポーネント = 1責務
Header・Footer・Card等、役割を明確に
🔁
再利用できるように設計
propsで柔軟にカスタマイズ可能にする
📦
小さく保つ
50行を超えたら分割を検討する
⬆️
データは上から下へ（props）
親→子の一方向データフロー
