セキュリティ：公開前に必ず確認しよう

SQLインジェクション
ユーザー入力をそのままSQLに埋め込むと危険。悪意ある入力でDBが破壊される。
❌ `WHERE id=${req.params.id}`
✅ prepare('...WHERE id=?').run(id)
APIキーの漏洩
コードにAPIキーを直書きしてGitHubにpushすると即座に不正利用される。
❌ apiKey = "sk-abc123..."
✅ apiKey = process.env.API_KEY
入力バリデーション
全てのユーザー入力を検証しよう。長さ・型・必須チェックを必ず実装する。
if (!text || text.length > 100) {
  return res.status(400).json({error})
}
HTTPS 必須
VercelもRailwayも自動でHTTPS化してくれる。HTTPのAPIは絶対に使わない。
✅ https://my-api.railway.app
❌ http://my-api.railway.app
デプロイ前チェック
✓.gitignore に .env を含む
✓全APIにバリデーションがある
✓SQLはプリペアドステートメント使用
Claude Code に聞けること

•「バリデーションが不足している箇所を教えて」
