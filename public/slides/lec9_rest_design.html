REST API の設計ルール

URLで「何を」、メソッドで「どうする」を表す

メソッドURL例操作
GET/usersユーザー一覧を取得
GET/users/42ID=42のユーザー取得
POST/users新しいユーザーを作成
PUT/users/42ID=42のユーザーを更新
DELETE/users/42ID=42のユーザーを削除
GET/users/42/postsそのユーザーの投稿一覧

REST APIの5原則

リソースはURLで表す

/getUserではなく /users/42 と名詞で表現する

HTTPメソッドで操作を表す

GET/POST/PUT/DELETEで意図を明確にする

JSON形式でやり取り

リクエストとレスポンス両方をJSON形式にする
