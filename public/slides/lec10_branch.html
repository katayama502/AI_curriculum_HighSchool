ブランチとマージ

ブランチ＝開発の分岐点

main
← 新機能を取り込んだ！
feature
新機能を実装
# ブランチを作って移動 $ git branch feature/login $ git checkout feature/login # または1行で $ git checkout -b feature/login # ブランチ一覧を確認 $ git branch # mainに戻ってマージ $ git checkout main $ git merge feature/login

ブランチを使う理由

mainを守る

mainブランチは常に動く状態を保つ。機能開発は別ブランチで行う。

並行開発できる

AさんはログインUI、BさんはAPI連携を同時に開発できる。

失敗しても安全

ブランチを削除すれば実験的な変更をなかったことにできる。

💡 命名規則の例：
feature/ログイン機能
fix/ボタン押せないバグ
hotfix/本番の緊急修正
