テストの種類：3つのレベル

単体テスト
Unit Test
関数1つが正しく動くかをテスト。速くて安定している。最も数が多い。
例：足し算関数 add(2,3) が 5 を返すか
統合テスト
Integration Test
複数のモジュールが正しく連携するかをテスト。APIエンドポイント全体を確認。
例：POST /api/todos がDBに保存して201を返すか
E2Eテスト
End-to-End Test
ブラウザを操作してユーザー体験を再現するテスト。最も現実的だが時間がかかる。
例：Playwrightでボタンクリックから結果表示まで確認
単体テスト
多数・高速・安価
統合テスト
中程度
E2Eテスト
少数・低速・高コスト
