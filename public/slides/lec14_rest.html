📐 RESTful APIの設計原則
# ユーザーリソースのREST API例

GET /api/users # 全ユーザー取得
GET /api/users/:id # 特定ユーザー取得
POST /api/users # 新規ユーザー作成
PUT /api/users/:id # ユーザー更新
DELETE /api/users/:id # ユーザー削除

# レスポンス例

✅ REST API設計のルール
📝
URLはリソース名（名詞）を使う
/getUsers ❌ → /users ✅
🔢
HTTPメソッドで操作を表す
GET=取得 POST=作成 PUT=更新 DELETE=削除
📊
ステータスコードを正しく使う
200=成功 201=作成 404=未発見 500=エラー
📦
JSONでデータをやり取りする
リクエスト・レスポンスともにJSON形式
