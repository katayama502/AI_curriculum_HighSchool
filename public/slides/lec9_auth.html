APIキーと認証

APIキーの使い方

# APIキーをヘッダーに含める（Python） import requests import os API_KEY = os.environ.get("MY_API_KEY") headers = { "Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json" } res = requests.get(url, headers=headers)

やってはいけないこと！

APIキーをソースコードに直接書いてGitHubに公開すると、他人に使われて高額請求になることがあります。絶対にNG！

安全な管理方法

環境変数（.envファイル）に保存してos.environ.get()で読み込む。.envファイルは.gitignoreに追加！

主な認証方式

APIキー

最も一般的。ヘッダーまたはクエリパラメータで送る。Claude APIもこの方式。

Bearer Token（JWT）

ログイン後に発行されるトークン。有効期限があり、より安全性が高い。

OAuth 2.0

「Googleでログイン」など。ユーザーが許可した範囲のみAPIを使える高度な仕組み。

APIキー不要

Open-Meteo（天気）など公開APIは認証なしで使える。学習に最適！
