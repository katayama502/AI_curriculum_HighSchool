パフォーマンス最適化をAIと行う

よくある最適化パターン

✗ 毎回APIを呼ぶ（遅い）
✓ キャッシュを使う（速い）
// ボタンを押すたびにAPI呼び出し
btn.addEventListener('click',
async () => {
const data = await fetch(url);
display(data);
}
);
// キャッシュがあれば再利用
const cache = {};
async function getWeather(city) {
if (cache[city]) return cache[city];
const data = await fetch(url);
cache[city] = data;
return data;
}

最適化の依頼パターン

キャッシュの追加
同じデータを何度もAPIから取得しないようにする
script.jsにキャッシュを追加して。同じ都市のデータは5分間は再取得しない。
デバウンスの実装
キーボード入力のたびにAPIを呼ばないようにする
検索入力のイベントにdebounceを追加して。入力後500ms待ってからAPIを呼ぶ。
ローディング表示
API待ち中はローディングスピナーを表示する
API呼び出し中はローディングアニメーションを表示して。完了したら非表示に。
レスポンシブ対応
スマートフォンでも見やすいレイアウトにする
style.cssをモバイル対応にして。スマホ画面（375px以下）でも使いやすいレイアウトに。
